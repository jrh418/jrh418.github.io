blt_textfield / bltTextField

The bltTextField component is used to collect text or numeric input from the user. This component is essentially a combination of the standard HTML <input> and <textarea> elements along with the incorporation of a label. The bltTextField component provides a clean and simple way to collect free text data.

Best Practice The data- prefix is not required for ngBoltJS attributes, but is highly recommended to prevent conflicts with standard HTML attributes.

Dependencies

  • BltApi
  • $timeout
  • $rootScope.Scope

Attributes

Param Type Details
data-name string

This attribute indicates the name of this form element and will be used during form traversal by the ngBoltJS framework.
data-label string

This attribute specifies the label for this component.
data-autofocus boolean

Indicates whether or not this field should autofocus on page load.
data-change expression

This attribute is used to bind an expression in the containing scope that will be invoked any time the value of this component changes. Functionality is based on the Angular ngChange directive.
data-disabled boolean

Disables the field. Any value set in this attribute will cause the field to be disabled.
data-max number

The maximum numeric value of the component. This is only valid with data-type="number". It will be ignored in all other cases.
data-maxlength number

The maximum number of characters to accept as valid in this component.
data-min number

The minimum numeric value of the component. This is only valid with data-type="number". It will be ignored in all other cases.
data-minlength number

The minimum number of characters to accept as valid in this component.
data-model expression

This attribute is used to bind the value of this component to a property in the containing scope. Functionality is based on the Angular ngModel directive.
data-pattern string

Defines a pattern to use in validation of this component's value. Can be any valid regular expression.
data-required boolean

Indicates whether or not this field is required.
data-rows number

The number of rows to present in a multi-line text area. This is only valid with data-type="textarea". It will be ignored in all other cases.
data-step number

Defines the intervals to use when changing the value of a number component. This is only valid with data-type="number". It will be ignored in all other cases. See W3C documentation on step for more information.
data-type string

This attribute defines the type of data that will be collected in this component. All HTML input types are technically supported, but the intended use case of this component includes the following standard types: text, password, tel, email, number, and url with the addition of textarea. Will default to text if no value is defined.
data-validate expression

An expression that gets passed through to an instance of the bltValidate directive to invoke custom validation on this component value. See documentation for bltValidate for more information.
data-autocomplete boolean

Indicates whether or not this field should autocomplete.
data-autocorrect boolean

Indicates whether or not this field should have autocorrect.
data-spellcheck boolean

Indicates whether or not this field should have spellcheck.
data-tabindex number

Specifies the tab order of an element.
