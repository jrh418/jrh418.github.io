blt_panel / bltPanel

since 1.0.0

The bltPanel component is used for displaying menus, forms, small tables, and related content. Panels consist of two auto-generated elements: .panel-overlay and .panel. The .panel-overlay fades in and covers and darkens the application window when the panel is open. The .panel slides in from the right, top, left, or bottom. The presentation of the panel can be customized through the attributes, class elements, and modifiers supported on the component.

Best Practice Keep the HTML template of your panel in an HTML partial file and load into the .panel-content element using ng-include.

You may optionally assign a controller for your panel on the .panel-content element as well.

Best Practice Place your panel as a direct child of the main.template.html and give the panel its own controller.

Dependencies

  • BltApi
  • $timeout

Attributes

Param Type Details
id string

Required in order for you to open and close the panel using the BltApi pub/sub methods.
data-position string

You can set the position of your panel. Valid values are: right, left, top, and bottom.

default: right
data-fixed string

If the panel is nested inside a bltView, set to true to expand panel to the full height (or width if position is top or bottom) of the application window. The panel expands to the constraints of its parent.

CSS Classes

Param Type Details
panel-content element

The element that contains your code for the panel. Add this inside the blt-panel component opening tag and closing tags. Use ng-include to load the rest of your panel content from an HTML partial. Use on an HTML <div>, <section>, or <form> tag.
panel-body element

The main content of the panel. The .panel-body does not have padding by default. To add padding, use the .panel-wrapper modifier. Should be used on the HTML <div> or <section> tags.
panel-wrapper modifier

The .panel-body has no padding by default as most components to be placed in a panel (such as forms and menus) require no padding. To add padding around text elements, add the .panel-wrapper class to the text element or on a containing <div>.
panel-header element

The header of the panel containing the .panel-title and optional buttons or icons. Should be used on an HTML <header> tag.
panel-footer element

Contains action buttons for the panel. Should be used on an HTML <footer> tag. By default, has no top-border. Menus, lists, forms, or tables used in a panel often have a border and another border on the footer would be redundant. If a border is needed, use the .edge-top utility class.
panel-title element

The title of the panel. Must be a child of the .panel-header element and should be used on an HTML <h2> tag.
panel-heading element

A heading element for sectioning content in the .panel-body. Should be used on an HTML <h3> tag. Add a .panel-wrapper class on the element to add padding around the heading unless the .panel-body already has .panel-wrapper on it.
panel-text element

Default text for the panel. Use this on any text content inside the .panel-body. Should be used on the HTML <p> tag. If there is no .panel-wrapper on the .panel-body, use the .panel-wrapper class on the text to add padding.
panel-divider element

A thick horizontal border for sectioning .panel-body content. Only use if not using a .panel-heading. Should be used on an HTML <hr> tag.
panel-btn-text element

Default text button for panel secondary actions. Should be used on the HTML <button> tag. For primary actions use .panel-btn-text-submit.
panel-btn-icon element

Icon button for panels, most often used in the .panel-body. Should be used on the HTML <button> tag.
panel-btn-solid element

Default solid button for panel secondary actions. Should only be used in the .panel-body. Should be used on the HTML <button> tag. For primary actions, use .panel-btn-solid-submit.

Most of the time, you will want to keep the markup of your .panel-content in a separate file and load it as a partial. This is easily done using ng-include on the .panel-content element.

ngBoltJS has a custom directive for easily opening and closing panels and other components. Simply pass the id of the panel you'd like to open or close. 


    
    

    // If the close button is inside the modal component, you do not have to pass the id of the modal.

You may need to do some other work/setup in your code before opening the panel. In that case, you can use the pub/sub methods of the BltApi to open and close the panel instead of the blt-open and blt-close directives. When closing a panel, include any functions to reset data in a $timeout to run after the closing animation of the panel and to avoid resetting data and 'flashing' the panel content as it animates out. 


    angular.module('myApp')
       .controller('PanelController', PanelController);

    // NOTE: You must inject the ngBoltJS API service into the controller to open and close panels via the API.
    PanelController.$inject = ['BltApi', '$timeout'];

    function PanelController(bltApi, $timeout) {
      var ctrl = this;
      ctrl.panel = { title: 'My Panel', activeMenu: 1 };
      ctrl.openMyPanel = function(){
        // Do some work here first...

        // When ready, publish the id of the panel you wish to open
        bltApi.publish('myPanel', 'open');
      }

      ctrl.closeMyPanel = function(){
        // publish the id of the panel you wish to close
        bltApi.publish('myPanel', 'close');

        // reset panel data after the closing animation
        $timeout(function(){
            ctrl.panel.title = '';
            ctrl.panel.activeMenu = -1;
        }, 350);
      }
    }

MANUAL SIZING: To customize the width (vertical panels) or height (horizontal panels), change the CSS width (or height) property of the panel component by using its id in your app.scss file. 


    // Vertical Panels
    #myRightPanel{
        .panel{
            width: 12.5rem; //200px
        }
    }

    // Horizontal Panels
    #myTopPanel{
        .panel{
            min-height: 10rem; //160px
        }
    }

FORMS AND PANELS: You may want to have a small form in a panel. In order to utilize default AngularJS functionality of validating and submitting forms, you must use the form tag on the .panel-content element.
