blt_core / BltApi

Application-wide API which provides a communication bus between components, navigating the view, generating unique ids, and defining the log levels. Most ngBolt components will inject the Bolt API and it is recommended that all controllers, factories, and components for specific applications also inject the Bolt API.

subscribe(topic, callback);

Summary

Subscribes to a topic and registers a callback function to be executed when a message is received.

Description

This is useful when needing to set up lines of communication between controllers and directives, or when needing to complete some function upon completion of an asynchronous function. Subscriptions should be made in the activate method of a controller's function.

Subscription callback methods are not watched by Angular, therefore any changes to the model must be wrapped in a $timeout so that they are digested by Angular and the view is updated.

Parameters

Param Type Details
topic string

The name of the subscription. You can subscribe more than one callback to the same name.
callback function

The function for handling the published message.

Example


  // Create A Controller
  angular.module('bltDocs')
    .controller('MyController', MyController)
  ;

  /**
   * @ngdoc controller
   * @name My Controller
   * @module bltDocs
   * @description An example controller showing how to subscribe to the Bolt API for messages
   * @param {BltApi} api Bolt API
   * @param timeout Angular $timeout
   * @constructor activate()
   */
  function MyController(api, $timeout) {
    // Bind To Controller
    var ctrl = this;

    // Activate Controller On Init
    ctrl.$onInit = activate();

    // Subscribe To The Bolt API For Messages
    function activate() {
      // Subscribe To 'test' Messages
      api.subscribe('test', function(message) {
        // NOTE: This Function Is Called When Any Bolt API Message Is Received

        // Check The Message
        // NOTE: This Message Could Be Any Type That The Publisher Sent
        if ( message == 'new') {
          // Do Some Logic
          // NOTE: We Wrap The Logic In A $timeout To Trigger Angular's Digest Cycle
          $timeout(function() {
            ctrl.currentItem = {};    // Not Implemented In This Example
            ctrl.createNewItem();     // Not Implemented In This Example
          });
        }
      }  // End Of api.subscribe
    }  // End Of activate()
  }  // End Of MyController

  // Inject Controller Dependencies
  MyController.$inject = [
    'BltApi',
    '$timeout'
  ];

unsubscribe(topic, callback);

Summary

Removes all callback functions registered to a subscription topic.

Description

The unsubscribe method removes all subscriptions of the provided name. When the scope of a controller or directive is destroyed, it is considered best practice to unsubscribe any subscriptions made in that controller or directive. They will be re-subscribed when they are added back to the scope.

The unsubscribe method will delete all subscriptions registered to a particular name regardless if that subscription was made from the controller or directive being destroyed. To prevent the unwanted deletion of subscriptions made to the same name but from a different controller/directive, use unique names on subscriptions made from different controllers or directives.

Parameters

Param Type Details
topic string

The name of the subscription you wish to delete.
callback function

An optional callback function to run after the subscription has been deleted.

Example


  // Create A Controller
  angular.module('bltDocs')
    .controller('MyController', MyController)
  ;

  /**
   * @ngdoc controller
   * @name My Controller
   * @module bltDocs
   * @description An example controller showing how to unsubscribe from messages from the Bolt API
   * @param {BltApi} api Bolt API
   * @param timeout Angular $timeout
   * @param $scope Angular $scope
   * @constructor activate()
   */
  function MyController(api, $timeout) {
    // Bind To Controller
    var ctrl = this;

    // Activate Controller On Init
    ctrl.$onInit = activate();

    // Subscribe To The Bolt API For Messages
    function activate() {
      // Subscribe To 'test' Messages
      api.subscribe('test', function(message) {
        // NOTE: This Function Is Called When Any Bolt Message Is Received

        // Check The Message
        // NOTE: This Message Could Be Any Type That The Publisher Sent
        if ( message == 'new') {
          // Do Some Logic
          // NOTE: We Wrap The Logic In A $timeout To Trigger Angular's Digest Cycle
          $timeout(function() {
            ctrl.currentItem = {};    // Not Implemented In This Example
            ctrl.createNewItem();     // Not Implemented In This Example
          });
        }
      }  // End Of api.subscribe

      // Call Destroy Function When Controller Is Destroyed
      $scope.$on('$destroy', destroy);
    }  // End Of activate()

    // Clean Up The Controller When Destroyed
    function destroy() {
      // Unsubscribe From Subscriptions
      api.unsubscribe('test', function() {
        // NOTE: This Function Is Called When The Unsubscribe Was Successful

        // Log Success
        api.log("Successfully Removed All Subscriptions For 'test' Topic!");
      });

      // Delete Controller From Scope
      // NOTE: This Is Only Needed If You Use controllerAs In The HTML Template
      delete $scope.MyCtrl;
    } // End Of destroy()
  }  // End Of MyController

  // Inject Controller Dependencies
  MyController.$inject = [
    'BltApi',
    '$timeout',
    '$scope'
  ];

publish(topic, message);

Summary

Publishes a message to the Bolt API, which is then received by subscribers to the same topic.

Description

The publish method provides a built-in ability to interact with ngBolt components from within a controller or custom directive as well as send custom messages between ngBolt application modules.

Parameters

Param Type Details
topic String

The name of the subscription you are publishing to. If opening a modal or panel, this will be the ID of the modal or panel, otherwise it will be the name you passed to BltApi.subscribe.
message String,Object

A string or object to be passed to the callback of the subscription you are publishing to. If opening a modal or panel, message will be 'open'. Likewise, if closing a panel or modal, it will be 'close'. If publishing to a custom subscription, this will be the string or object you expect to handle.

Example


  // Create A Controller
  angular.module('bltDocs')
    .controller('MyController', MyController)
  ;

  /**
   * @ngdoc controller
   * @name My Controller
   * @module bltDocs
   * @description An example controller showing how to publish a message to the Bolt API
   * @param {BltApi} api Bolt API
   * @param timeout Angular $timeout
   * @constructor activate()
   */
  function MyController(api, $timeout) {
    // Bind To Controller
    var ctrl = this;

    // Activate Controller On Init
    ctrl.$onInit = activate();

    // Subscribe To The Bolt API For Messages
    function activate() {
      // Open A Modal With An ID Of 'logout'
      api.publish('logout', 'open');

      // Close A Panel With An ID Of 'editUser'
      api.publish('editUser', 'close');

      // Publish A Custom Message To A Custom Subscriber
      var message = {
        action: 'edit',
        data: {
          name: 'Foo'
        }
      }
    }  // End Of activate()
  }  // End Of MyController

  // Inject Controller Dependencies
  MyController.$inject = [
    'BltApi',
    '$timeout'
  ];

switchViews(path, searchParams);

Summary

A helper function for updating Angular's $location.path() and/or $location.searchParams().

Description

Updates $location with the provided path and search params. If only path is provided, $location.path() will be called, if searchParams are included, $location.url() is called. If searchParams is an emtpy object, any search parameters on the current url will be removed.

If the current URL has search parameters but only path is passed as a parameter, only the path will change and the search parameters will still be applied to the URL. This can be useful when switching views when filtering by some value(s).
See Routes & Views for more information on defining views.

Parameters

Param Type Details
path string

The path of the view you want to open.
searchParams object

Search parameters to append to the supplied path. If the object is empty, search parameters will be removed.

Example


  // Create A Controller
  angular.module('bltDocs')
    .controller('MyController', MyController)
  ;

  /**
   * @ngdoc controller
   * @name My Controller
   * @module bltDocs
   * @description An example controller showing how to switch views
   * @param {BltApi} api Bolt API
   * @constructor activate()
   */
  function MyController(api) {
    // Bind To Controller
    var ctrl = this;

    // Activate Controller On Init
    ctrl.$onInit = activate();

    // Switch View Examples
    function activate() {
      // NOTE: The Following Example Assume The Base URL Is http://example.com/

      // Change The URL To:
      //   HTML5: http://example.com/some/path
      //   Non-HTML5: http://example.com/#!/some/path
      api.switchViews('/some/path');

      // Change The URL To:
      //   HTML5: http://example.com/some/path?foo=value1&bar=value2
      //   Non-HTML5: http://example.com/#!/some/path?foo=value1&bar=value2
      api.switchViews('/some/path', {foo: 'value1': bar: 'value2'});

      // If The Current URL Is: http://example.com/some/path?foo=value1&bar=value2
      // Change The URL To:
      //   HTML5: http://example.com/some/other/path
      //   Non-HTML5: http://example.com/#!/some/other/path
      api.switchViews('some/other/path', {});

      // If The Current URL Is: http://example.com/some/path?foo=value1&bar=value2
      // Change The URL To:
      //   HTML5: http://example.com/some/other/path?foo=value1&bar=value2
      //   Non-HTML5: http://example.com/#!/some/other/path?foo=value1&bar=value2
      api.switchViews('some/other/path');

      // If The Current URL Is: http://example.com/some/path?foo=value1&bar=value2
      // Change The URL To:
      //   HTML5: http://example.com/some/path
      //   Non-HTML5: http://example.com/#!/some/path
      bltApi.switchViews(undefined, {});
    }  // End Of activate()
  }  // End Of MyController

  // Inject Controller Dependencies
  MyController.$inject = [
    'BltApi'
  ];

setLogLevel(level);

Summary

Sets application-wide console logging level.

Description

Sets the application-wide log level, affecting the logs that are printed to the console when invoking any of the following BltApi logging functions: trace, debug, info, warn and error. The levels are listed in order of precedence from lowest to highest. When the log level is set, the log levels that are enabled include that specific level and any level of higher precedence. If you set the level to 'info', the levels enabled will be info, warn and debug.

The log level that is assigned will remain in affect until the user clears it via the BltApi method. This will persist across browser and user sessions.

Parameters

Param Type Details
level number

The log level to assign.

Returns

Type Details
boolean

Whether or not the given log level was successfully applied.

getMillisFromEpoch(date);

Summary

Converts date to milliseconds from epoch.

Description

Attempts to convert the given date parameter to milliseconds from epoch. Acceptable formats include:

  • number - will be interpreted as ms from epoch.
  • string - will attempt to interpret as a formatted string, or a string containing ms from epoch if that fails
  • date - will be converted to ms from epoch using date.getTime()
  • function - will convert the output of the given date function to ms from epoch.
  • null - if no parameter is provided, will return current ms from epoch. new Date().getTime()

Parameters

Param Type Details
date Object,Number,String,function()

The date or date-like object to convert to ms from epoch.

Returns

Type Details
number

Milliseconds from epoch.
