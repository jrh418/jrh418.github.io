Application Structure

All ngBolt applications share a common folder / file structure which allows the CLI to configure and run the app. When a new ngBolt app is created using the CLI, it will produce the following folders / files.

  • my-app
  • config
  • ng-bolt-app
  • package.json
Folder / File Description
my-app This is the root of the ngBolt application
config All configuration relating to the ngBolt project (build/deploy, styles, routes, views, etc.)
ng-bolt-app All of the source code for the specific ngBolt application
package.json Description of the ngBolt application
There are more files in an ngBolt app but we will only discuss the most important files in this section. You can view the template project for a full file list.

Configuration

The ngBolt CLI relies on JSON configuration files for building, running, and deploying applications. These configuration files are all located in the config folder at the root of the application. The section below describes the function for each file within this configuration section.

  • config
  • profiles
    • development.json
  • build.json
  • routes.json
  • styles.json
  • views.json
Folder / File Documentation Description
profiles Bolt CLI A single profile tells ngBolt which Data API to use, what Data Source to use, as well as any overrides to the build.json file. Use the Bolt CLI to select a profile when running or deploying an ngBolt application (it will default to development.json if none is specified).
build.json Build & Deploy Used by the Bolt CLI to know how to build & package the application (ports, additional libraries, documentation, packaging, ngBolt components, etc.).
routes.json Data API Used by the Data API to understand how to request and parse data from different sources (i.e. REST vs WAMP vs SQL).
styles.json Styles & Variables Used to configure the styles (i.e. colors) of the individual ngBolt Components.
views.json Bolt API A wrapper around ngRoute, which identifies and links controllers / templates to the URL / parameters.

ngBolt App

This folder contains all the application code. By default, the template will also include a core and main folder with some pre-configured application logic.

As you progress through the development of your application, it is highly recommended that you refer to the Angular 1 Style Guide by John Papa for recommendations on project file structure and Angular best practices.
  • ng-bolt-app
  • core
    • core.module.js
    • shared.factory.js
  • images
    • favicon.png
    • logo.png
  • main
    • main-appbar.template.html
    • main.controller.js
    • main.module.js
    • main.template.html
  • app.js
  • app.scss
  • index.html
Folder / File Required Description
core This folder is used to hold any shared factories used between multiple modules within the application.
core.module.js Add any shared factories to this module.
shared.factory.js Inject this module into other modules to share data between them when needed.
images Override the login icon as well as the favicon. You can also add any additional images to include in the deployment to this folder.
main This is where all the main application code lives. This folder should not be added to, just modified as needed. Create other folders in the ng-bolt-app for other modules and functionality.
main-appbar.template.html This is the main appbar that's available for newly created applications. It can be removed if unused.
main.controller.js The top level controller for a ngBolt app which should only contain logic that's shared across every other controller in the application. It is aliased and available to templates as Main.
main.template.html The main template for your application. This will almost always contain a reference to the view (see the Bolt API for how to setup and use views).
app.js This is the top level angular application which defines the ngBolt app. Inject angular dependencies here as needed.
app.scss This is the top level stylesheet for an ngBolt application. It typically only imports the stylesheets defined in the other modules within the project.
index.html This is the index page for the application. You shouldn't need to update this file.
The Bolt CLI relies on the following files: app.js, app.scss, index.html, main.controller.js, main.module.js, and main.template.html. All other files can be removed as applicable.
