Command Line Interface

The ngBolt CLI can setup a new ngBolt project, run an ngBolt application in a development environment and deploy to a production environment.

Please refer to Step 1 in the Getting Started section for details on how to install the ngBolt CLI.

Usage

bolt [command] [profile] [flags]

Commands

New

To create a new project, run the following command (replacing my-app with your app name) in the directory where you want to create your new project:

bolt new [my-app]

Follow the on-screen instructions to complete the app creation.

Run

While inside your project's folder, host your application using the specified profile:

bolt run [profile]
The ngBolt CLI will look in the config/profiles folder for a file named profile.json. If no profile is specified, the development profile will be used.

Deploy

While inside your project's folder, compress and package your application using the specified profile:

bolt deploy [profile]
The ngBolt CLI will look in the config/profiles folder for a file named profile.json. If no profile is specified, the development profile will be used.

Flags

Flag Description Valid Input
-h, --help Output CLI usage information (including commands and flags)
-V, --version Output the version number for the ngBolt CLI
-f, --fatal [value] The error level that will exit the build process error (default), warning, or off
-p, --platform [value] The platform that the app will be deployed to angular (default), rails, grails, or cordova
-e, --env [value] The environment for the gulp build development (default) or production
-s, --serve [boolean] Run a server as part of a gulp build (this is default behavior when platform is angular) true or false
-b, --beautify [boolean] Prevent minification of assets true or false
