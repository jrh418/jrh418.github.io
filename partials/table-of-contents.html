Getting Started

The Getting Started section is a Quick Start guide to installing ngBolt and creating a new application using the CLI.

Tutorial

The tutorial will walk new developers through creating a new application using the CLI. Developers will learn how to configure new applications, add modules, access data from multiple sources, design for multiple screen sizes, and package the application in multiple formats.

By the end of the tutorial, developers should feel comfortable with the CLI as well as have a general understanding of an ngBolt project.

Step Page Description
1 Introduction A brief introduction to ngBolt and the CLI
2 Configuration Using config files to configure ngBolt components
3 To Do Module Creating a module for viewing To Do items
4 To Do Editor Creating a form for adding and updating To Do items
5 Data Access Saving To Do items to local storage & HTTP endpoints using the Data API
6 Responsive Design Updating the UI for multiple screen sizes
7 Build & Deploy Packing an application for use on the web and on a handheld

Fundamentals

The fundamentals section provides a base knowledge which will help developers use ngBolt to create applications. It builds on the information from the tutorial by going more in-depth into ngBolt architecture, components, and tools.

By reading through the fundamentals section, a developer should have all the knowledge needed to build complex ngBolt applications.

Page Description
Architecture A deep dive into what ngBolt provides, how it works, and some of the use cases
Command Line Interface A full description of available CLI commands as well as how to use them
Application Templates When ngBolt creates a new application, a template is cloned and updated with the user's input
Application Structure How every ngBolt project is set up (files, configuration, etc.) and the available configuration options
Authentication How to use the Authentication API across authentication methods
Routes & Views How to define routes and views for application workflows
Grid Layout Flexbox is used as the underlying grid layout with a few small additions
Styles & Variables Styles and variables are used as input to ngBolt applications to automatically style them using a light / dark theme
Form Controls All supported form elements are available as ngBolt components
UI Components Additional user interface components available to developers
Data API Accessing data from multiple sources depending on the environment
Build & Deploy Applications can be packaged and deployed in different ways depending on the target environment

Best Practices

The Best Practices section goes into details on how ngBolt applications should be developed. These aren't required, but build on best practices from various other groups (i.e. Angular, Javascript, etc.).

The intention of this section is to make developers aware of the appropriate style guidelines from all the tools / languages used by ngBolt.

Page Description
Documentation ngBolt uses dgeni for documentation and will automatically add user documentation from the ngBolt applications into the packaged documentation
Style Guide ngBolt uses Google's Material Design specification for components and developers should continue to use that specification when creating applications
Responsive Design ngBolt supports the developer when creating responsive applications by allowing styles / sections to show / hide based on screen size
Testing ngBolt uses a combination of karma, chi, mocha, sinon, and protractor for unit tests which can also be used by application developers
Going To Production ngBolt offers additional flags using the CLI to package applications into a production ready format

## Examples There are three applications that ngBolt has for examples. Each app tries to utilize different fundamentals while sticking to the style guides. Developers should review each application but should keep in mind these examples don't fully cover the ngBolt capability and should only be used for reference.
Page Description
Places App A simple application for viewing locations on a map and information about those places. Uses Cesium for maps, ArcGIS for tiles, and Open Street Maps for data.
Chat App An application for chatting between users in groups. Uses Crossbar.io (WAMP) for cross-client communication.
To Do App A web and handheld application for tracking to do items. This is also used as the tutorial for ngBolt.
